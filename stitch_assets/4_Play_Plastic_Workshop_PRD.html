# Play-Plastic Workshop ## Product Overview **The Pitch:** An educational playground disguised as a toy mechanic's garage, where fixing engines and balancing tires secretly teaches arithmetic, logic, and pattern recognition. **For:** Children aged 5-8 who learn better through tactile construction and "helping out" than through abstract quizzes. **Device:** Desktop (Touch-optimized for hybrid laptops/large touchscreens) **Design Direction:** "Play-Plastic" — A 2.5D aesthetic where every element looks like a high-quality, injection-molded plastic toy. Soft bevels, rounded corners, and inner shadows create a tactile feel without heavy 3D rendering. **Inspired by:** Duplo, Toca Boca, classic Fisher-Price "Garage" playsets. --- ## Screens - **[Main Garage Hub]:** The central workshop menu where children choose which machine to fix. - **[Gearbox Station]:** A shape-sorting and logic puzzle interface. - **[Tire Balance Bay]:** A math-focused station using weights and scales. - **[Wiring Panel]:** A pattern-matching interface for logical connections. - **[Fuel Mix Monitor]:** A liquid-fill interface teaching addition and volume. - **[Victory Modal]:** The reward screen displayed after completing a repair job. --- ## Key Flows **[Flow: Balancing a Tire]:** A user practices simple addition to balance an equation. 1. User is on **[Main Garage Hub]** -> clicks the wobbling car on the lift. 2. User enters **[Tire Balance Bay]** -> sees a wheel needing 4 more "weight units." 3. User drags a "2" block and a "2" block from the **[Tool Tray]** onto the rim. 4. **[Result]**: The blocks snap with a plastic "clack," the wheel spins smoothly, and the user earns a star. **[Flow: Adaptive Hinting]:** The system detects a struggle and simplifies the UI. 1. User is on **[Gearbox Station]** -> repeatedly tries to jam a Square Gear onto a Triangle Peg. 2. After 3 failures -> **[Reduced Complexity Mode]** triggers. 3. Distractor gears fade out -> The correct Triangle Peg pulses with a soft yellow glow. 4. **[Result]**: User successfully places the gear, receiving positive reinforcement ("Good persistence!"). ---
Design System ## Color Palette - **Primary Blue:** `#4DA6FF` - Garage walls, primary buttons (Sky Blue) - **Base Grey:** `#E5E7EB` - Floor concrete, neutral backgrounds - **Toy Red:** `#FF5C5C` - Wires, critical interactive parts (Softened, not alarmist) - **Toy Yellow:** `#FFD93D` - Hints, gears, highlights - **Toy Green:** `#4ADE80` - Success states, fuel, go buttons - **Surface Wood:** `#FDBA74` - Workbench tops (Warm beige-orange) - **Plastic Dark:** `#4B5563` - Tires, text, deep shadows - **Shadow/Depth:** `rgba(0,0,0,0.15)` - Used for "inset" and "drop" shadows to create plastic volume ## Typography Fonts must be rounded, friendly, and highly legible. - **Headings:** 'Fredoka One', 700, 32-48px (Rounded, bubbly) - **Body:** 'Varela Round', 400, 18px (Clean, soft terminals) - **Numbers/Data:** 'Nunito', 800, 24px (High legibility for math) - **Labels:** 'Fredoka One', 400, 14px (All caps, tracking 0.5px) **Style Notes:** All text containers have `border-radius: 12px` and a `box-shadow: inset 0 -2px 0 rgba(0,0,0,0.1)` to look like stickers applied to plastic. ## Design Tokens ```css :root { --color-plastic-blue: #4DA6FF; --color-plastic-red: #FF5C5C; --color-plastic-yellow: #FFD93D; --color-plastic-green: #4ADE80; --color-surface-wood: #FDBA74; /* The "Toy" look is defined here */ --shadow-plastic-up: inset 0 4px 6px rgba(255,255,255,0.4), 0 4px 0 rgba(0,0,0,0.1); --shadow-plastic-down: inset 0 4px 8px rgba(0,0,0,0.15); --shadow-plastic-pressed: inset 0 2px 4px rgba(0,0,0,0.2); --radius-xl: 24px; --radius-lg: 16px; --font-heading: 'Fredoka One', cursive; --font-body: 'Varela Round', sans-serif; } ```

Screen Specifications ### [Main Garage Hub] **Purpose:** The "Overworld" where the child selects a mission. **Layout:** Full-screen illustration of a workshop interior. Top HUD fixed. Bottom interactive area is the floor/lift. **Key Elements:** - **Car Lift (Center):** Large red hydraulic lift holding a cartoon car. Car bounces gently on hover. - **Workbench (Right):** Wooden desk with scattered gears. Click to enter Gearbox Station. - **Tire Rack (Left):** Stack of tires against the wall. Click to enter Tire Balance Bay. - **Top HUD:** 80px height. White pill-shape container. Holds "Back" arrow (left) and "Star Count" (right). **States:** - **Default:** Car is dirty/broken (smoke puff animation). - **Completed:** Car is shiny/clean. Lift is lowered. **Interactions:** - **Hover Station:** The object (e.g., Tire Stack) scales up 1.05x and glows white. - **Click Station:** Camera zooms into that area (CSS Scale transform) before page transition. --- ### [Gearbox Station] **Purpose:** Shape recognition and logic (Sorting). **Layout:** 1. **Top HUD:** "Fix the Gears" title. 2. **Main Stage:** An open metal gearbox panel with empty shaped pegs. 3. **Bottom Tray:** Wooden texture "drawer" holding loose gears. **Key Elements:** - **The Panel:** `#E2E8F0` metallic grey background. Inset holes (dark inner shadow). - **Target Pegs:** Raised visual style. Triangle, Square, Circle shapes. - **Draggable Gears:** Brightly colored (Pink, Cyan, Lime). Thick borders to look like chunky plastic. - **Conveyor Belt (Advanced):** Animated strip bringing new gears in on higher levels. **Interactions:** - **Drag Start:** Gear lifts (shadow moves down/right), scales 1.1x. - **Drop (Correct):** "Clack" sound, gear snaps to center of peg, green ring expands from center. - **Drop (Incorrect):** "Boing" sound, gear slides back to tray using spring physics. **Components:** - **Gear Object:** 88x88px, SVG vector, `drop-shadow(0 4px 0 rgba(0,0,0,0.2))` for thickness. --- ### [Tire Balance Bay] **Purpose:** Basic Arithmetic (Addition/Subtraction). **Layout:** Split screen. Left side is the car axle. Right side is the weight supply. **Key Elements:** - **The Wheel:** Large black rubber tire with a silver rim. It wobbles visibly. - **Digital Readout:** An LED-style display on the axle showing "Target: 10" and "Current: 0". - **Weight Blocks:** Rectangular lead weights labeled "1", "2", "5". Metallic texture. - **Rim Slots:** Indented rectangular zones on the wheel rim to accept weights. **States:** - **Unbalanced:** Wheel tilts and shakes. Readout is Red. - **Balanced:** Wheel straightens. Readout turns Green. Wheel spins rapidly. **Responsive:** - **Desktop:** Side-by-side (Wheel Left, Weights Right). - **Tablet:** Vertical (Wheel Top, Weights Bottom). --- ### [Wiring Panel] **Purpose:** Pattern Recognition and Logic. **Layout:** A circuit board fills the center screen. **Key Elements:** - **Sockets:** Circular ports with colored rings (Red, Blue, Yellow). - **Wires:** Bezier curves drawn between touch points. - **Spark Gap:** A broken fuse area in the middle. **Interactions:** - **Drag Wire:** User touches a "Source" socket and drags. A thick rope-like line follows the finger/cursor. - **Snap Connection:** When released near a matching "Destination" socket, the wire snaps and pulses light. - **Error:** Connecting Red to Blue causes a harmless "fizz" spark and the wire retracts. **Adaptive Difficulty:** - **Guided Mode:** The correct destination socket pulses when the user picks up a wire. --- ### [Fuel Mix Monitor] **Purpose:** Volume and Fractions/Ratio. **Layout:** A transparent fuel tank marked with measurement lines (1-10). **Key Elements:** - **The Tank:** Glass texture `rgba(255,255,255,0.3)` with white tick marks. - **Jerry Cans:** 3 distinctive cans (Green=1 unit, Orange=2 units, Purple=3 units). - **Pour Animation:** Clicking a can tilts it; liquid flows; tank level rises with a bubbling animation. - **Overflow Valve:** If user pours too much, liquid spills out a side pipe (playful fail state). **Interactions:** - **Click Can:** Can rotates 45deg, liquid particle stream emits. - **Level Rise:** The fluid inside the tank is a `div` with `height` transition easing-out. --- ### [Victory Modal] **Purpose:** Reward and transition. **Layout:** Overlay modal on top of the current station. **Key Elements:** - **Star Burst:** 3 animated gold stars pop in sequence (Pop... Pop... Pop!). - **Mascot Message:** A robot mechanic says "Great Job!" or "Engine Fixed!". - **Buttons:** - **Primary:** "Next Job" (Green, Right Arrow icon). - **Secondary:** "Back to Shop" (Blue, Garage icon). **Components:** - **Card Container:** White surface, rounded-xl, huge drop shadow. - **Confetti:** CSS particle explosion behind the modal.

Build Guide **Stack:** HTML5, Tailwind CSS v3, JavaScript (Vanilla or lightweight framework like Alpine.js for state). **Build Order:** 1. **Design Tokens & Global CSS:** Define the "Plastic" look (shadows, bevels) first. If this isn't right, nothing looks like a toy. 2. **Components Library:** Build the `DraggableItem`, `DropZone`, and `PlasticButton` components. These are reused everywhere. 3. **Main Garage Hub:** Establishes the navigation structure. 4. **Gearbox Station:** Easiest logic to implement (Drag & Drop matching). 5. **Tire Balance Bay:** Introduces math logic. 6. **Adaptive System:** Layer the hint logic on top of existing games.
